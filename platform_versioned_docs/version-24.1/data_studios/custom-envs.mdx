In addition to the Seqera-provided container template images, you can provide your own custom container environments, either by augmenting the Seqera-provided images with a list of Conda packages that you provide or by providing your own base container template image.

## Conda packages

### Prerequisites

- Wave must be configured. For more information, see [Wave containers][wave].
 
 :::note
 Enterprise deployments must either allow access to the public Wave server, or self-host their own Wave server.
 :::

### Conda package syntax {#conda-package-syntax}

When adding a new data studio session, you can customize its configuration to install a list of Conda packages to the template image. The supported schema is identical to that used by the Conda `environment.yml` environment file. For more information on the Conda environment file, see [Creating an environment file manually][env-manually].

```yaml title="Example environment.yml file"
name: example-env
channels:
  - defaults
  - conda-forge
dependencies:
  - python=3.7
  - numpy
  - pandas
  - scikit-learn
  - pip:
    - matplotlib
    - seaborn
    - jupyter
```

To create a data studio with custom Conda packages, see [Add a data studio][add-ds].

## Custom container template image {#custom-containers}

For advanced use cases, you can build your own container template image.

:::note
Amazon Elastic Container Registry (ECR) is the only supported private container registry.
:::

### Prerequisites

- Access to a container image repository, either a public container registry or a private Amazon ECR repository
- A container template image

### Dockerfile configuration {#dockerfile}

For your custom template container image, you must use a Seqera-provided base image and include several additional build steps for compatibility with Data Studios.

#### Ports

The container must use the value of the `CONNECT_TOOL_PORT` environment variable as the listening port for whatever interactive software that you include in your custom container.

#### Signals

Upon termination, the container's main process must handle the `SIGTERM` signal and perform any necessarily cleanup. After a 30 second grace period, the container receives the `SIGKILL` signal.

#### Minimal Dockerfile

The minimal Dockerfile includes directives to accomplish the following:

- Pulls a Seqera-provided base image with prerequisite binaries
- Installs prerequisite packages, such as wget and fuse
- Installs and configures Seqera Fusion
- Sets the container entry points

Customize the following Dockerfile to include any additional software that you require:

```docker title="Minimal Dockerfile"
# Seqera base image
FROM public.cr.seqera.io/platform/connect-client:0.7.1 as connect

# 1. Add connect binary
COPY --from=connect /connect-client /usr/bin/connect-client

# 2. Install required system packages
RUN apt-get update --yes && apt-get install --yes --no-install-recommends \
    ca-certificates \
    wget \
    fuse \
    btrfs-progs \
    squashfs-tools \
    && apt-get clean && rm -rf /var/lib/apt/lists/*

# 3. Install Fusion
RUN wget https://fusionfs.seqera.io/releases/pkg/2/4/2/fusion-amd64.tar.gz && \
    tar xvzf fusion-amd64.tar.gz && \
    rm fusion-amd64.tar.gz

# 4. Configure Fusion env variables
ENV FUSION_LOG_OUTPUT=/tmp/.fusion.log
ENV FUSION_COMMAND_BUFFER_SIZE=0
ENV FUSION_VALIDATE_WORK=false

# 5. Configure connect and Fusion as the entrypoint
ENTRYPOINT ["/usr/bin/fusion", "/usr/bin/connect-client"]
```

For example, to run a basic Python-based HTTP server, build a container from the following Dockerfile. When a data studio runs the custom template environment, the value for the `CONNECT_TOOL_PORT` environment variable is provided dynamically.

```docker title="Example Dockerfile with Python HTTP server"
FROM public.cr.seqera.io/platform/connect-client:0.7.1 as connect

FROM ubuntu:20.04
RUN apt-get update --yes && apt-get install --yes --no-install-recommends python3

COPY --from=connect /connect-client /usr/bin/connect-client

RUN apt-get update --yes && apt-get install --yes --no-install-recommends \
    ca-certificates \
    wget \
    fuse \
    btrfs-progs \
    squashfs-tools \
    && apt-get clean && rm -rf /var/lib/apt/lists/*

RUN wget https://fusionfs.seqera.io/releases/pkg/2/4/2/fusion-amd64.tar.gz && \
    tar xvzf fusion-amd64.tar.gz && \
    rm fusion-amd64.tar.gz

ENV FUSION_LOG_OUTPUT=/tmp/.fusion.log
ENV FUSION_COMMAND_BUFFER_SIZE=0
ENV FUSION_VALIDATE_WORK=false

ENTRYPOINT ["/usr/bin/fusion", "/usr/bin/connect-client"]

CMD ["/usr/bin/bash", "-c", "python3 -m http.server $CONNECT_TOOL_PORT"]

```

To create a data studio with a custom template image, see [Add a data studio][add-ds].
