# MultiQC — troubleshooting placement

Placement rule for **MultiQC** docs. Apply on top of `../core/troubleshooting.md`.

## Where it goes

**MultiQC has no dedicated troubleshooting pages in this repo.** Do **not** move
troubleshooting content off a MultiQC page — there is no destination for it.
Leave it inline: keep it as a `## Troubleshooting` section (or a `:::note` for a
single limitation), and still apply the core "Format on the destination page"
rules to the entry so it reads consistently.

If MultiQC later adds a troubleshooting page or directory, record the destination
and the existing-vs-new rule here, and this product will start moving content like
the others. Until then, leave inline and, if in doubt, ask the user.
