# Nextflow — troubleshooting placement

Placement rule for **Nextflow** docs. Apply on top of `../core/troubleshooting.md`.

## Where it goes

**Nextflow documentation lives outside this repo** (the open-source Nextflow docs
have their own conventions), so `docs-structure` has no troubleshooting destination
for it here. Do **not** move troubleshooting content — leave it in place, and apply
the core "Format on the destination page" rules to the entry in situ.

If a Nextflow doc in this repo needs troubleshooting relocated, ask the user where
it should go rather than guessing.
