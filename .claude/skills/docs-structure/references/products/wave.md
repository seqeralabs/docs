# Wave — troubleshooting placement

Where troubleshooting content goes for **Wave** docs. Apply on top of
`../core/troubleshooting.md`.

## Where it goes

Wave keeps a **single** troubleshooting page:

| Destination                            | Use                                    |
| -------------------------------------- | -------------------------------------- |
| `wave_docs/wave_repo/docs/troubleshoot.md` | All Wave troubleshooting topics        |

## Existing page or new page

There is one page, so there is no "new page" choice: **add** each topic as a `##`
section to `troubleshoot.md`, following the page's existing pattern. Don't create
a per-feature troubleshooting page for Wave unless the user asks.

If you can't confirm the current Wave troubleshooting layout, ask before moving.
