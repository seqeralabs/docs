# Fusion — troubleshooting placement

Where troubleshooting content goes for **Fusion** docs. Apply on top of
`../core/troubleshooting.md`.

## Where it goes

| Destination                     | Use                                                       |
| ------------------------------- | --------------------------------------------------------- |
| `fusion_docs/troubleshooting/`  | Feature-scoped troubleshooting pages, when the area has one |
| `fusion_docs/troubleshooting.md`| The single top-level troubleshooting page                 |

## Existing page or new page

- **Add to an existing page** — add the topic as a `##` section to the page that
  covers the area, or to `fusion_docs/troubleshooting.md` when no feature-scoped
  page fits. This is the common case.
- **Create a new page** in `fusion_docs/troubleshooting/` only when a feature has
  enough entries to warrant its own page; add it to the sidebar.

If you can't confirm the current Fusion troubleshooting layout, ask before moving.
