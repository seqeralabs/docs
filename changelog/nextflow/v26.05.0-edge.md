## Feature updates and improvements

### Language features

- Removed the deprecated `echo` process directive by @pditommaso in [#7206](https://github.com/nextflow-io/nextflow/pull/7206)
- Added support for record type input with dot-notation CLI params by @jorgee in [#7170](https://github.com/nextflow-io/nextflow/pull/7170)

### Azure

- Improved Azure Batch VM image diagnostics and fixed stale 24.04 docs by @pditommaso in [#7200](https://github.com/nextflow-io/nextflow/pull/7200)

### Plugins

- Added workflow DAG reporting to the `nf-tower` plugin by @ewels in [#7218](https://github.com/nextflow-io/nextflow/pull/7218)
- Allowed the plugin version to be specified when running a plugin command by @JosuaCarl in [#7113](https://github.com/nextflow-io/nextflow/pull/7113)
- Added subcommand listing to `nextflow plugins -help` by @ewels in [#7197](https://github.com/nextflow-io/nextflow/pull/7197)

### General

- Added `-help` as an alias for `-h` in top-level CLI options by @ewels in [#7238](https://github.com/nextflow-io/nextflow/pull/7238)
- Added GPU metrics to the workflow-level execution summary by @cristianrcv in [#7229](https://github.com/nextflow-io/nextflow/pull/7229)
- Added hidden plural/singular aliases for noun subcommands by @ewels in [#7239](https://github.com/nextflow-io/nextflow/pull/7239)
- Added Nextflow module info to the lineage `TaskRun` entry by @jorgee in [#7160](https://github.com/nextflow-io/nextflow/pull/7160)
- Added `seqera://` data-links support to the `nf-tower` filesystem by @jorgee in [#7070](https://github.com/nextflow-io/nextflow/pull/7070)
- Added a `shellEnabled` option and per-task prediction model to the Seqera executor by @pditommaso in [#7204](https://github.com/nextflow-io/nextflow/pull/7204)
- Added the `nextflow logfile` command by @ewels in [#7177](https://github.com/nextflow-io/nextflow/pull/7177)
- Added a data lineage architecture decision record (ADR) by @bentsherman in [#7187](https://github.com/nextflow-io/nextflow/pull/7187)
- Changed `stdout` outputs to `stderr` to avoid breaking piped output by @ewels in [#7121](https://github.com/nextflow-io/nextflow/pull/7121)
- Clarified the distinction between modules and script inclusion in the docs by @bentsherman in [#7217](https://github.com/nextflow-io/nextflow/pull/7217)
- Migrated the docs to the new build by @christopher-hakkaart in [#7119](https://github.com/nextflow-io/nextflow/pull/7119)
- Improved Tower telemetry error handling on transient gateway failures by @pditommaso in [#7190](https://github.com/nextflow-io/nextflow/pull/7190)
- Mirrored test containers to `public.cr.seqera.io` to harden CI against quay.io outages by @pditommaso in [#7195](https://github.com/nextflow-io/nextflow/pull/7195)
- Promoted the `latest` stable channel by version comparison instead of branch name by @pditommaso in [b5ac62e0c](https://github.com/nextflow-io/nextflow/commit/b5ac62e0c)
- Refactored `changelog.txt` to `CHANGELOG.md` by @ewels in [#7179](https://github.com/nextflow-io/nextflow/pull/7179)
- Added support for the accelerator directive in the local executor by @bentsherman in [#5850](https://github.com/nextflow-io/nextflow/pull/5850)
- Used a stop flag instead of a thread interrupt to shut down the logs checkpoint by @pditommaso in [#7193](https://github.com/nextflow-io/nextflow/pull/7193)

- Fixed lineage records for stub-run and eval outputs by @jorgee in [#7211](https://github.com/nextflow-io/nextflow/pull/7211)
